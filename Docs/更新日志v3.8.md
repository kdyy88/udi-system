# 更新日志 v3.8

日期：2026-03-16

## 版本概述

v3.8 是账号体系与权限的全面重构版本。核心目标是将原先依赖 `user_id` 查询参数的信任模型，升级为基于 HTTP-only JWT Cookie 的标准认证体系，并引入邮箱注册、验证、密码重置等完整用户自助流程。

---

## 一、后端 — 认证基础设施（Phase 1–2）

### 1) fastapi-users 接入

- 引入 `fastapi-users[sqlalchemy]>=13.0.0`，使用 `CookieTransport` + `JWTStrategy`。
- Cookie 名称：`udi_auth`，`HttpOnly=true`，`SameSite=lax`，JWT 有效期 7 天。
- `User` 模型继承 `SQLAlchemyBaseUserTable[int]`，新增 `email`、`is_active`、`is_verified`、`is_superuser` 字段。
- 新增 `db/user_manager.py`：自定义 `UserManager`，支持 SHA-256 旧密码向 bcrypt 的懒迁移。
- 新增 `db/fastapi_users_config.py`：集中管理认证后端、依赖项 `current_active_user`、`current_admin_user`。

### 2) 数据库迁移

- 新增 Alembic 迁移 `0006_add_user_auth_fields.py`：
  - 添加 `email`（唯一索引）、`is_active`、`is_verified`、`is_superuser` 列。
  - 为已有用户自动填充初始 email（`username@system.local`），admin 用户自动设为已验证。

### 3) 认证路由重写

- `api/auth.py` 重写，挂载全套 fastapi-users 路由：
  - `POST /auth/jwt/login` / `logout`
  - `POST /auth/register`、`/auth/forgot-password`、`/auth/reset-password`
  - `POST /auth/request-verify-token`、`/auth/verify`
  - `GET /auth/users/me`
- 保留兼容旧前端的 `POST /auth/login`（JSON 格式），成功后同样签发 Cookie，并返回 `{user_id, username, email, role}`。

### 4) 邮件服务

- 新增 `services/email_service.py`，基于 **Resend SDK v2**（`Resend(key).emails.send()`）。
- 支持验证邮件（`send_verification_email`）和密码重置邮件（`send_reset_email`）。
- `RESEND_API_KEY` 为空时自动进入开发模式，邮件链接直接打印到后端日志。
- 发件人默认使用 `onboarding@resend.dev`（无需域名验证即可测试）。

### 5) Redis 限流基础设施

- 新增 `db/redis.py`：异步连接池，提供 `get_redis()` FastAPI 依赖。
- 支持 OTP 冷却（60 秒）+ 每日配额（10 次）限流。
- **启动时 Redis 不可达不再致命**：仅记录 warning，服务正常启动；限流端点返回 HTTP 503。
- `main.py` lifespan 对应调用 `init_redis()` / `close_redis()`。

## 二、后端 — RBAC 与行级数据隔离（Phase 3–4）

### 6) 全面替换 `user_id` 查询参数

- `api/labels.py`、`api/batches.py`、`api/templates.py`、`api/system.py` 所有端点移除 `user_id: Query` 参数。
- 用户身份统一由 `user: User = Depends(current_active_user)` 从 JWT Cookie 中提取，从根本上杜绝越权访问。
- 管理员专属操作（system 端点）改为 `Depends(current_admin_user)`，非 admin 用户返回 HTTP 403。

### 7) Schema 清理

- `LabelCreateRequest` 移除 `user_id: int` 字段。
- `BatchCreateRequest` 移除 `user_id: int` 字段。
- `LegacyLoginRequest` / `LegacyLoginResponse`（含 `email` 字段）新增至 `schemas/label.py`。

## 三、后端 — 依赖与配置

### 8) pyproject.toml 调整

- 新增生产依赖：`fastapi-users[sqlalchemy]`、`PyJWT`、`bcrypt`、`redis`（v7 已内置 asyncio，移除 `[asyncio]` extra）、`resend`、`python-dotenv`。
- `anyio` 移至 `[dependency-groups] dev`（仅供 `@pytest.mark.anyio` 测试使用，不是生产依赖）。

### 9) 配置与 .env 加载

- `core/config.py` 新增 `REDIS_URL`、`JWT_SECRET`、`JWT_LIFETIME_SECONDS`、`RESEND_API_KEY`、`RESEND_FROM_EMAIL`、`FRONTEND_URL` 配置项。
- 集成 `python-dotenv`：本地 `uv run uvicorn` 启动时自动从项目根目录 `.env` 加载变量，无需手动 export。
- 新增 `.env`（gitignore）和 `.env.prod.example`（含 JWT_SECRET、RESEND 配置说明）。
- `docker-compose.yml` 通过变量插值（`${RESEND_API_KEY:-}`）读取 `.env`。

## 四、前端 — Cookie 认证适配（Phase 5）

### 10) API 层改造

- `lib/api.ts`：全局添加 `withCredentials: true`，新增 401 响应拦截器（自动清除本地认证状态并重定向至 `/login`）。
- `lib/auth.ts`：新增 `initSession()` —— 向 `GET /api/v1/auth/users/me` 查询当前 Cookie 对应的用户，刷新 localStorage 展示数据。
- `types/udi.ts`：`AuthUser` 和 `LoginResponse` 新增 `email` 字段。
