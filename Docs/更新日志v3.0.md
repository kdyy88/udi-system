# 更新日志 v3.0 — 工业级批量打码通道

> 发布日期：2025-07  
> 基于 v2.1

---

## 总体目标

将系统从"单标签逐条生成"升级为"工业级批量打码通道"，支持 Excel 批量上传、
客户端并行渲染、JSZip 一键打包、历史批次管理与钻取。

---

## 新增功能

### 1. Excel 批量上传

**文件**：`frontend/lib/excelParser.ts`、`frontend/app/batch/page.tsx`

- 支持 `.xlsx` / `.xls` 文件拖拽上传或点击选择
- 自动检测表头（中英文均可）并映射字段；无表头时按列位置回退
- 列映射：GTIN-14 / 批次号 / 有效期 / 序列号 / 生产日期 / 备注
- 客户端 GTIN-14 格式校验（14 位纯数字）
- 单次最多 500 行，无效行高亮展示，跳过空行
- 一键下载 Excel 模板（无服务端依赖，xlsx 在浏览器生成）

### 2. 批量保存 API

**文件**：`backend/app/api/batches.py`、`backend/app/schemas/batch.py`

- `POST /api/v1/batches/generate` — 单事务：创建 `LabelBatch` + 批量写入 `label_history`
  - 所有 HRI / GS1 字符串由后端 `gs1_engine.py` 权威重算，客户端不可信数据不写入
  - 最多 500 行，超限返回 400
  - 行级错误溯源：`Row {n} (GTIN {x}): {reason}`
- `GET /api/v1/batches` — 游标分页批次列表
- `GET /api/v1/batches/{id}` — 批次详情 + 标签游标分页（升序，用于 ZIP 重排）
- `DELETE /api/v1/batches/{id}` — 验证所有权，CASCADE 删除全部 `label_history`

### 3. Web Worker + Comlink 渲染引擎

**文件**：`frontend/workers/barcodeWorker.ts`、`frontend/lib/svgTemplates.ts`

- `BarcodeWorkerAPI.generateBatch(batchId, userId, template, onProgress)` 通过 Comlink 暴露
- 分页拉取 `GET /batches/{id}`（page_size=200）确保 500 条不阻塞主线程
- 每条记录使用 `@bwip-js/browser` 渲染：
  - DataMatrix (`gs1datamatrix`, scale 3)
  - GS1-128 线性码 (`gs1-128`, scaleX 2)
- `svgTemplates.ts` 为纯字符串函数（无 DOM/React），可在 Worker 中安全调用
- 两种标签模板：
  - **compact**（30×30 mm）— 仅 DataMatrix + 元数据文字
  - **dual**（80×30 mm）— DataMatrix + GS1-128 + HRI 文字
- ZIP 文件命名：`{GTIN-14}_{序列号|序号:04}.svg`

### 4. JSZip + file-saver 一键下载

**文件**：`frontend/hooks/useBatchUpload.ts`

- `jszip` 在 Worker 内将所有 SVG 打包为 DEFLATE 压缩 ZIP
- `file-saver saveAs(blob, filename)` 在主线程下载（避免 Safari content URL 限制）
- ZIP 文件命名：`UDI_{批次名}_{batch_id}.zip`
- 进度回调通过 `Comlink.proxy()` 跨 Worker 边界传递，主线程实时更新进度条

### 5. 批量打码页面（状态机）

**文件**：`frontend/app/batch/page.tsx`、`frontend/hooks/useBatchUpload.ts`

六阶段状态机：`idle → parsing → validated → saving → generating → done | error`

- **idle**：展示拖拽上传区 + 下载模板按钮
- **parsing**：正在解析 Excel，loading 指示器
- **validated**：展示行表格（有效/无效统计）+ 模板选择器 + "保存并生成"按钮
- **saving**：调用 `POST /batches/generate`
- **generating**：实时进度条（n / total）
- **done**：成功提示 + 跳转批次详情链接 + "新建批次"按钮
- **error**：详细错误信息 + 重置按钮

### 6. 批次详情页

**文件**：`frontend/app/history/batch/[id]/page.tsx`

- 路由：`/history/batch/[id]`
- 展示批次元信息（名称、来源、总数、创建时间）
- 分页标签列表（page_size=50，正序，支持下一页）
- **重新下载 ZIP** 按钮：在页面内新建 Worker，重新渲染并下载同批次 ZIP

### 7. 历史台账双 Tab

**文件**：`frontend/app/history/page.tsx`

- 新增 `批次总览` Tab（默认）：展示批次列表（名称/来源/数量/日期/详情/删除）
- 原 `全部明细` Tab：保留原有单条记录表格及筛选功能
- 批次 Tab 支持游标分页（`useLabelBatches` hook）
- 删除批次 = CASCADE 删除所有子记录，二次确认弹窗

### 8. 导航栏更新

**文件**：`frontend/components/shared/Navbar.tsx`

- 新增导航链接：标签生成 / 批量打码（`/batch`）/ 历史台账

## 数据库变更

### 新表：`label_batch`

```sql
CREATE TABLE label_batch (
    id          SERIAL PRIMARY KEY,
    user_id     INTEGER NOT NULL REFERENCES users(id),
    name        VARCHAR(200) NOT NULL,
    source      VARCHAR(20)  NOT NULL,   -- 'excel' | 'form'
    total_count INTEGER      NOT NULL,
    created_at  TIMESTAMPTZ  NOT NULL
);
CREATE INDEX ix_label_batch_id ON label_batch (id);
CREATE INDEX ix_label_batch_user_id ON label_batch (user_id);
CREATE INDEX ix_lb_user_id_desc ON label_batch (user_id, id);
```

### 变更：`label_history`

```sql
ALTER TABLE label_history
    ADD COLUMN batch_id INTEGER REFERENCES label_batch(id) ON DELETE CASCADE;
CREATE INDEX ix_lh_batch_id ON label_history (batch_id);
```

> Alembic 迁移文件：`backend/alembic/versions/0002_add_label_batch.py`
