### 5) Cursor-based 分页（替代 offset/limit）

**背景**：`offset/limit` 在大数据量下需要数据库扫描跳过前 N 行，即使有索引开销也随页数增大。
Cursor 分页通过 `WHERE id < :cursor ORDER BY id DESC` 每次都从索引起点开始扫描，性能稳定。

**利用已有索引**：`ix_lh_user_id_desc (user_id, id)` — v2.0 已建立，本次开始真正利用。

**后端变更：`backend/app/schemas/label.py`**

```python
# 移除
class LabelHistoryListResponse(BaseModel):
    total: int
    page: int        # ← 删除
    page_size: int   # ← 删除
    items: list[LabelHistoryResponse]

# 替换为
class LabelHistoryListResponse(BaseModel):
    total: int
    next_cursor: int | None  # ← 新增（None 表示末页）
    items: list[LabelHistoryResponse]
```

**后端变更：`backend/app/api/labels.py`**

| 参数 | v2.0 | v2.1 |
|------|------|------|
| `page: int = 1` | 有 | **删除** |
| `page_size: int = 10` | 有 | 保留 |
| `cursor: int \| None = None` | 无 | **新增** |

查询逻辑变化：
```python
# v2.0（offset 扫描）
.order_by(LabelHistory.created_at.desc(), LabelHistory.id.desc())
.offset((page - 1) * page_size)

# v2.1（cursor 索引直扫）
.where(LabelHistory.id < cursor)   # 仅当 cursor 非 None
.order_by(LabelHistory.id.desc())  # 走 ix_lh_user_id_desc
```

COUNT 查询保持不含 cursor filter（始终反映"共 N 条"总数）。

**前端类型变更：`frontend/types/udi.ts`**
```ts
// 移除 page, page_size；新增 next_cursor
type LabelHistoryListResponse = {
  total: number;
  next_cursor: number | null;
  items: LabelHistoryItem[];
};
```

**前端 Hook 变更：`frontend/hooks/useLabelHistory.ts`**

| 能力 | v2.0 | v2.1 |
|------|------|------|
| 内部状态 | `const [page, setPage] = useState(1)` | `const [cursorStack, setCursorStack] = useState<(number \| null)[]>([null])` |
| 当前页标识 | `page` 数字 | `cursorStack` 末尾元素（null = 首页） |
| 下一页 | `setPage(p => p + 1)` | `setCursorStack(prev => [...prev, data.next_cursor!])` |
| 上一页 | `setPage(p => Math.max(1, p - 1))` | `setCursorStack(prev => prev.slice(0, -1))` |
| 筛选重置 | `setPage(1)` | `setCursorStack([null])` |
| 对外接口 | `page, pageSize` | `hasPrev: boolean, hasNext: boolean` |

**DataTable / LabelHistorySection / 两个页面：** 同步更新 pagination prop 类型，翻页按钮 disabled 条件由 `page <= 1` / `page >= totalPages` 改为 `!hasPrev` / `!hasNext`，底部状态栏移除"第 X/Y 页"显示（cursor 分页不维护当前页码）。

- `backend/app/schemas/label.py`
- `backend/app/api/labels.py`
- `frontend/types/udi.ts`
- `frontend/hooks/useLabelHistory.ts`

### 7) Lint warnings 全清

**`@next/next/no-img-element` 警告（共 5 处）**

这些 `<img>` 标签均渲染 `data:image/svg+xml;...` URI，`next/image` 不支持 data URI，因此不转换为 `<Image>`，逐处添加 eslint-disable 注释并注明原因：

| 文件 | 位置 | 原因 |
|------|------|------|
| `components/labels/PreviewTemplateCanvas.tsx` | `renderBarcode()` 函数 × 2 | data URI（SVG）不支持 next/image |
| `components/labels/PreviewTemplateCanvas.tsx` | dual 模板 × 2 | 同上 |
| `components/labels/BarcodePreview.tsx` | 第 18 行 | data URI（PNG base64）不支持 next/image |

**`react-hooks/set-state-in-effect` warning（1 处）**

在 `app/page.tsx` 的 `useEffect` 中 `setAuthUser(user)` 添加 eslint-disable 注释（auth 状态初始化场景，无级联渲染风险）。

**`react-hooks/exhaustive-deps` warning（1 处）**

在 `app/history/page.tsx` 的 `useEffect` 中添加 eslint-disable 注释并说明：`router` 为 Next.js 稳定引用，state setter 同样稳定，刻意只在 mount 时运行一次。
