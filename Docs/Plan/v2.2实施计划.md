### 阶段一：数据库 + 后端（无前端依赖，优先完成）

1. **Alembic 迁移** — 新建 `backend/alembic/versions/0002_add_label_batch.py`
   - 新建 `label_batch` 表：`id PK`, `user_id FK→users`, `name String(200)`, `source String(20)`（`"excel"`/`"form"`）, `total_count Integer`, `created_at DateTime TZ`
   - `label_history` 新增 `batch_id Integer FK→label_batch NULLABLE`
   - 新增索引 `ix_lh_batch_id (batch_id)`
   - 存量 `batch_id=NULL` 数据在历史"全部明细"Tab 继续可见，不受影响

2. **ORM 更新** — models.py
   - 新增 `LabelBatch` model
   - `LabelHistory` 新增 `batch_id: Mapped[int | None]` 列，带 `nullable=True` FK

3. **批次 Schema** — 新建 `backend/app/schemas/batch.py`
   - `BatchCreateRequest`: `{ user_id, name, items: list[LabelInput] }`（前端仅发原始元数据，不含 hri）
   - `BatchCreateResponse`: `{ batch_id, name, source, total_count, created_at }`
   - `LabelBatchSummary`: `{ id, name, source, total_count, created_at }`
   - `LabelBatchListResponse`: `{ total, next_cursor: int|None, items: list[LabelBatchSummary] }`
   - `LabelBatchDetailResponse`: `{ ...LabelBatchSummary, next_cursor: int|None, labels: list[LabelHistoryResponse] }`

4. **批次路由** — 新建 `backend/app/api/batches.py`（prefix `/batches`）

   | 路由 | 说明 |
   |------|------|
   | `POST /batches/generate` | 单一事务：创建 `LabelBatch` → `gs1_engine.py` 逐行重算 HRI/GS1 → `insert_many` label_history；失败时整体回滚 |
   | `GET /batches` | 批次列表，cursor 分页（`WHERE user_id=? [AND id < cursor] ORDER BY id DESC`） |
   | `GET /batches/{id}` | 批次详情 + 所含标签 cursor 分页（供详情页和 Worker 取 hri 列表） |
   | `DELETE /batches/{id}` | 验证 `user_id` 归属后，级联删除 label_history 再删 label_batch |

5. **`POST /generate` 联动** — 修改 labels.py
   - 单条录入时自动创建 `source="form", total_count=1` 的 `LabelBatch`，新 label_history 的 `batch_id` 指向它
   - 前端零改动，完全后端自动处理

6. **注册路由** — 修改 router.py，挂载 `batches` router

### 阶段三：前端核心库

9. **Excel 解析 + 校验** — 新建 `frontend/lib/excelParser.ts`
   - `parseExcel(file: File): Promise<RawRow[]>` — xlsx 解析，日期序列号 → `YY/MM/DD`
   - `validateRows(rows): ValidatedRow[]` — 调用 `validateGtin14()` + try/catch `buildGs1ElementString()`
   - 返回 `ValidatedRow = { status: "valid"|"error"; errorMessage?: string; rowIndex: number; data: LabelInput; hri?: string; gs1ElementString?: string }`

10. **SVG 纯函数模板** — 新建 `frontend/lib/svgTemplates.ts`
    - Worker 内无 DOM/React，需纯字符串拼接
    - `renderCompactSvg(datamatrixSvg, hri): string`
    - `renderDualSvg(datamatrixSvg, gs1128Svg, hri): string`
    - `renderDetailSvg(datamatrixSvg, gs1128Svg, hri): string`
    - 复用 `DUAL_BARCODE_WIDTH/HEIGHT` 等常量（从 `barcode-svg.ts` 导出）

11. **Web Worker（Comlink）** — 新建 `frontend/workers/barcodeWorker.ts`

    Worker 侧暴露 class（通过 `Comlink.expose()`）：
    ```ts
    class BarcodeWorker {
      async generateBatch(
        hriList: string[],          // 来自 GET /batches/{id} 的权威数据
        template: TemplateKey,
        onProgress: (done: number, total: number) => void  // Comlink.proxy()
      ): Promise<Blob>              // ZIP Blob
    }
    ```
    内部：50 条分片 → 每条 `bwipjs.toSVG()` ×2（DataMatrix + GS1-128）→ `svgTemplates` 拼合 → `jszip.file(name, svgString)` → 分片后 `onProgress` → 全部完成后 `jszip.generateAsync({type:"blob"})` 返回

    文件命名：`{GTIN}_{serialOrIndex:04}.svg`（从 hri 解析 AI 01 和 AI 21）

12. **批量上传 Hook** — 新建 `frontend/hooks/useBatchUpload.ts`

    状态机：`idle → parsing → validated → template-selecting → saving → done | error`

    注意 `saving` 阶段（步骤 3）在 `template-selecting` 之后、Worker 启动之前——先入库，获得 `batchId` 后再跳转详情页。

13. **批次历史 Hook** — 新建 `frontend/hooks/useLabelBatches.ts`
    - TanStack Query，queryKey `["label-batches", userId, cursor]`，cursor 分页，风格与 `useLabelHistory` 完全一致

**Decisions**
- Worker 用 `GET /batches/{id}` 的 hri 而非内存数据：详情页成为完全无状态的持久视图，随时可重复下载
- 后端 `POST /batches/generate` 拒绝 `hri`/`gs1_element_string` 字段（schema 层不接收），强制重算：捍卫后端权威性
- `svgTemplates.ts` 纯函数层与 `PreviewTemplateCanvas` 共享常量但不共享 JSX：Worker 安全，主线程 React 组件不受影响
- Comlink `proxy()` 传进度回调：避免 raw `postMessage` 的状态同步复杂度
- 批次删除级联：先 `DELETE FROM label_history WHERE batch_id=?` 再 `DELETE FROM label_batch`，单事务
