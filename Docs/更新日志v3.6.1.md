# 更新日志 v3.6.1

日期：2026-03-14

## 版本概述

v3.6.1 是一次小版本整理，重点修复批次链路中的正确性问题，并对前后端结构做轻量收敛，提升可维护性与文档一致性。

---

## 本次整理项

### 1) 批次链路修复

- 修复批次详情分页游标逻辑，统一为升序读取标签列表，避免翻页重复或漏数。
- 批量生成时为 `label_batch` 持久化 `template_definition` 快照。
- 历史批次页“重新下载 ZIP”时优先复用原批次模板，避免退回到兜底样式。
- 新增 Alembic 迁移：`0005_add_batch_template_snapshot.py`。

### 2) GS1 规则前后端对齐

- 前端 `lib/gs1.ts` 与后端 `gs1_engine.py` 对齐校验规则。
- 单标签与批量上传现在都要求：
  - DI 必须为合法 GTIN-14；
  - 至少填写一个 PI（批号 / 生产日期 / 有效期 / 序列号）。
- 同步更新前后端测试，避免规则再次漂移。

### 3) 前端结构整理

- 提取 `useRequireAuth()`，统一页面级登录态检查与退出逻辑。
- `AuthUser` 类型收敛为单一来源，避免重复定义。
- `PageHeader` 改为复用组件，减少页面壳重复代码。
- 清理无引用的前端死代码与过时 API 常量。

### 4) 后端 API 整理

- 新增 `backend/app/api/helpers.py`，集中处理：
  - 请求耗时日志；
  - 用户存在性校验；
  - 管理员权限校验；
  - owned-record 校验；
  - `LabelHistoryResponse` 序列化。
- `labels.py`、`batches.py`、`templates.py`、`system.py` 均已复用这些 helper，减少重复样板。

### 5) 文档同步

- 更新 `README.md` 与 `Docs/SOURCE_MAP.md`，移除过时描述。
- 文档现已反映：
  - 批次模板快照；
  - 前端统一认证守卫；
  - 后端 API helper 分层；
  - 当前分页与重下载行为。
