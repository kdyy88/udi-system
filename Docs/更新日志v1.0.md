# 更新日志 v1.0

## 版本目标
为当前 UDI 系统新增 SVG 条码生成能力，并在前端预览中支持 PNG/SVG 切换，确保可放大查看与矢量导出。

## 变更概览

### 1. 后端新增 SVG 预览接口
- 新增接口：`POST /api/v1/labels/preview-svg`
- 位置：backend/app/api/labels.py
- 能力：接收 `di` + PI 字段（`lot` / `expiry` / `serial`），返回：
  - `datamatrix_svg`
  - `gs1_128_svg`
  - `hri`
  - `gs1_element_string`

### 2. 新增 SVG 响应模型
- 位置：backend/app/schemas/label.py
- 新增模型：
  - `LabelPreviewSvgResponse`

### 3. 条码服务新增 SVG 生成函数
- 位置：backend/app/services/barcode_gen.py
- 新增函数：
  - `render_gs1_datamatrix_svg()`
  - `render_gs1_128_svg()`
- 说明：返回 SVG 字符串（矢量图内容），供前端直接渲染。

### 4. 前端类型扩展
- 位置：frontend/types/udi.ts
- 新增类型：
  - `LabelPreviewSvgResponse`

### 5. 前端预览弹窗支持 SVG 切换
- 位置：frontend/components/labels/PreviewDialog.tsx
- 变更：
  - 新增条码格式状态：`png | svg`
  - 选择 SVG 时调用 `/api/v1/labels/preview-svg`
  - 从 `HRI` 中提取 PI 字段，避免因缺少 PI 导致 400

### 6. 前端模板渲染支持 SVG data URI
- 位置：frontend/components/labels/PreviewTemplateCanvas.tsx
- 变更：
  - 按格式选择渲染：
    - PNG：`data:image/png;base64,...`
    - SVG：`data:image/svg+xml;utf8,...`（或兼容编码）

## 关键问题与修复记录

1. **404 问题**
- 现象：请求 `http://127.0.0.1:8000/labels/preview-svg` 返回 404
- 原因：缺少 `/api/v1` 前缀
- 修复：统一改为 `/api/v1/labels/preview-svg`

2. **400 问题**
- 现象：接口返回 `At least one PI value is required`
- 原因：仅传了 `di`，未传 PI 字段
- 修复：前端从 `HRI` 自动提取 `(10)/(17)/(21)` 并拼入请求

3. **前端“裂图”问题**
- 现象：SVG 在浏览器中渲染异常
- 原因：SVG data URI 编码方式不兼容
- 修复：调整为 UTF-8 data URI 编码并增加兼容处理

4. **试错遗留代码清理（本次）**
- 删除无调用接口：`POST /api/v1/labels/generate-svg`
- 删除无用模型：`LabelGenerateSvgResponse`（后端/前端）
- 删除前端无用参数：`PreviewTemplateCanvas` 的 `lot`
- 删除字段伪装映射：不再把 `datamatrix_svg/gs1_128_svg` 伪装成 `*_base64`
- 合并后端重复逻辑：抽取 GS1/HRI 构建与 SVG 生成公共函数
- 删除热循环内异常吞噬 `try/except pass`

## 接入步骤（给后续开发）

1. 后端：在 `barcode_gen.py` 增加 SVG 生成函数，并在 `labels.py` 暴露 `preview-svg` 路由。
2. 后端：在 `schemas/label.py` 增加 SVG 响应模型。
3. 前端：在 `types/udi.ts` 补充 SVG 类型。
4. 前端：在 `PreviewDialog.tsx` 增加格式切换 + SVG 接口调用。
5. 前端：在 `PreviewTemplateCanvas.tsx` 统一封装条码渲染函数，按格式输出 data URI。
6. 验证：
   - 生成标签后切换到 SVG 模式可正常显示
   - 网络请求为 `/api/v1/labels/preview-svg`
   - 返回数据包含 `datamatrix_svg` 与 `gs1_128_svg`

## 验收标准

- 接口不再出现 404/400。
- SVG 内容可放大不失真。
- PNG/SVG 双格式均可在同一预览流程中使用。
