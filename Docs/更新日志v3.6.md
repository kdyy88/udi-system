# 更新日志 v3.6

日期：2026-03-14

## 版本概述

v3.6 在 v3.5 可视化模板编辑器基础上，补全导航入口、单标签预览模板选择、GS1-128 比例约束、SVG 条码充满框架等一系列体验优化，并修复多个功能性 Bug。

---

## v3.6 新增 / 修复（本版本）

### 1) 管理员系统模板直接编辑（v3.6 最新）

#### 角色权限

- 登录接口 `POST /api/v1/auth/login` 响应新增 `role` 字段，前端 `AuthUser` 类型同步补充 `role: string`。
- 新增 `isAdmin(user)` 工具函数（`lib/auth.ts`），供各组件判断管理员身份并控制 UI 可见性。

#### 系统模板覆写后端

- 新增 `system_config` 表（Alembic 迁移 `0004_add_system_config.py`），Key-Value JSONB 结构：
  - `"hidden_system_templates"` → 被隐藏的系统模板 ID 数组。
  - `"system_template_overrides"` → 管理员修改的画布 JSON，Map<sys_id, CanvasDefinition>。
- 新增三个系统配置端点（`backend/app/api/system.py`）：
  - `GET  /api/v1/system/template-overrides` — 公开，返回所有画布覆写。
  - `PUT  /api/v1/system/template-override/{sys_id}?user_id=N` — 管理员保存/替换指定系统模板的画布。
  - `DELETE /api/v1/system/template-override/{sys_id}?user_id=N` — 管理员删除覆写，恢复出厂默认。

#### 前端覆写查询

- 新增 `hooks/useSystemTemplateOverrides.ts`：
  - `useSystemTemplateOverrides()` — 查询所有覆写（staleTime 60s）。
  - `useSaveSystemTemplateOverride(userId)` — Mutation：PUT 端点。
  - `useDeleteSystemTemplateOverride(userId)` — Mutation：DELETE 端点。
- 新增 `applyOverrides(overrides)` 辅助函数（`lib/systemTemplates.ts`）：将 DB 覆写合并到硬编码出厂模板数组，返回新数组，不改变原始常量。
- `TemplateGallery` 和 `PreviewDialog` 均使用 `applyOverrides(overrides)` 生成 `effectiveSystemTemplates`，确保用户始终看到最新版本。

#### 管理员编辑入口

- `/editor?seed=sys-xxx`：
  - 非管理员可访问，行为不变（加载系统模板画布用于参考）。
  - 管理员访问时：`editingSysId` 置为该 sys-id，模板名称**不加"副本"**（直接编辑本体）。
- 顶栏按钮逻辑（管理员 + `editingSysId` 不为 null）：
  - **「更新系统模板」**（主要）→ 调用 `PUT /api/v1/system/template-override/{sys_id}` → 保存成功后跳转 `/templates`。
  - **「另存为个人模板」**（次要）→ 行为与普通用户一致，创建个人 DB 模板。

#### TemplateGallery 管理员控制

- `TemplateGallery` 新增 `isAdmin?: boolean` prop（默认 `false`）。
- `mode="manage"` + `isAdmin=true` 时，系统模板卡悬浮显示三个按钮：
  - ✏️ **编辑**：链接至 `/editor?seed={id}`，在编辑器中直接修改本体。
  - 🔄 **恢复出厂默认**（仅在存在 DB 覆写时显示）：调用 DELETE 端点，amber 颜色提示有覆写。
  - 🙈 **隐藏/显示切换**：调用 `PUT /api/v1/system/hidden-templates`，隐藏后卡片对普通用户不可见。
- 管理员视图下，已隐藏模板以虚线边框 + 50% 透明度显示（管理员可见，普通用户不可见）。

涉及文件：
- `backend/app/api/system.py`（扩展：新增 3 个 override 端点）
- `backend/app/schemas/label.py`（改：`LoginResponse` 新增 `role: str`）

### 4) GS1-128 条碁6:1 比例约束

- 新增 `GS1_128_ASPECT_RATIO = 6`（宽:高）常量和 `isGs1128Type()` 工具函数；`DATAMATRIX_ASPECT_RATIO = 1`；`barcodeAspectRatio(type)` 统一辅助函数替代各处分散判断。
- **拖拽缩放**：`Canvas.tsx` 向 `<Rnd>` 传入 `lockAspectRatio={barcodeAspectRatio(el.barcodeType)}`，拖动任意角/边时另一维自动跟随。
- **属性面板手动输入**：改宽时自动推算高，改高时自动推算宽，面板显示“比例锁定 6:1”提示（DataMatrix 显示“比例锁定 1:1”）。
- **切换条码类型**：从 DataMatrix 切换至任意 GS1-128 时，自动以现有宽度重算高度使其符合比例。
- `makeBarcode` 初始默认尺寸：GS1-128 为 240×40px（恰好 6:1）；DataMatrix 保持 100×100px。

涉及文件：
- `frontend/stores/canvasStore.ts`（新增 `GS1_128_ASPECT_RATIO`、`isGs1128Type`，更新 `makeBarcode`）
- `frontend/components/editor/Canvas.tsx`（新增 `lockAspectRatio`）
- `frontend/components/editor/PropertiesPanel.tsx`（新增联动逻辑和提示）

### 5) SVG 条碼充满元素框架

- `embedSvg` 由 `<g transform="translate scale">` 改为嵌套 `<svg>` + `preserveAspectRatio="none"`。
- 条碼（DataMatrix / GS1-128）在导出 SVG 中完全填满用户绘制的元素框，不留白边。
- GS1-128 已有 6:1 比例约束，`none` 下不会产生视觉变形；DataMatrix 建议用户保持方形框。

涉及文件：
- `frontend/lib/svgTemplates.ts`（改：`embedSvg` 实现）

### 6) Bug 修复

| 问题 | 原因 | 修复 |
|---|---|---|
| 模板保存 404 | `router.py` 注册 templates 路由时附加了多余的 `prefix="/api/v1"`，与 `main.py` 的全局前缀叠加为 `/api/v1/api/v1/templates` | 删除 `router.py` 中的冗余 prefix |
| 编辑器页面布局溢出 | `h-screen` 未减去全局导航栏高度（56px），画布区域被压出视口 | 改为 `h-[calc(100vh-3.5rem)]` |

| `docker-compose exec backend alembic` 失败 | 在 `backend/` 子目录执行，且容器内 `alembic` 不在 `$PATH` | 改用本地 venv：`DATABASE_URL=...@localhost:5432 .venv/bin/alembic upgrade head` |

## 5) SVG 渲染引擎重写

- `frontend/lib/svgTemplates.ts` 完全重写：
  - 删除旧函数：`renderCompactSvg`、`renderDualSvg`、`renderDetailSvg`。
  - 新增 `renderCustomSvg(input: LabelSvgInput, canvas: CanvasDefinition): string`：
    - 遍历 `canvas.elements`，按 `type` 分发：
      - `barcode` → 调用 `pickBarcodeSvg`（bwip-js）生成真实条码 SVG，通过 `embedSvg` 嵌入。
      - `text` → 若有 `fieldBinding` 则通过 `findAiValue(hri, ai)` 填入实际值，否则显示静态 `content`。
      - `rect` → 直接输出 `<rect>` 元素。
  - 新增 `embedSvg(svg, ox, oy, maxW, maxH)` 辅助函数。
- 新增 `frontend/lib/gs1Utils.ts`，集中提供 `findAiText` / `findAiValue` / `escapeXml`，被多处共同引用。
- `batchExporter.ts` 的 `BatchExportOptions.templateDefinition: CanvasDefinition` 替代旧 `template: TemplateKey`。
- `useBatchUpload.ts` 的 `startGenerate` 参数由 `BatchTemplate`（字符串）改为 `CanvasDefinition`。

涉及文件：
- `frontend/lib/svgTemplates.ts`（重写）
- `frontend/lib/gs1Utils.ts`（新增）
- `frontend/lib/batchExporter.ts`（改造）

## 8) 已知限制 / v3.7 计划

- `history/batch/[id]` 重新下载 ZIP 时使用硬编码回退 `CanvasDefinition`（单 DataMatrix 元素），尚未关联原始模板。v3.7 将补充批次与模板的关联存储。

## 9) 部署说明

首次部署（或从 v3.5 以前升级）需执行数据库迁移。**注意**：迁移须在 `backend/` 目录下通过本地 venv 执行（容器内 `alembic` 不在 `$PATH`）：

```bash
cd /home/administrator/udi-system/backend
DATABASE_URL="postgresql+asyncpg://gs1user:gs1pass@localhost:5432/gs1udi" \
  .venv/bin/alembic upgrade head
# 应用 0003_add_label_template 迁移，创建 label_template 表
# 应用 0004_add_system_config 迁移，创建 system_config 表
# 确认当前版本：.venv/bin/alembic current  → 应输出 "0004 (head)"
```
