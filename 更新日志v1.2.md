# 更新日志 v1.2

日期：2026-03-10

## 本次完成（功能性更新优先）

### 1) 日期选择器改为 ShadcnUI 标准实现
- 将自定义 `input[type="date"]` 方案替换为 `Popover + Calendar` 标准组合。
- 展示格式统一为：`YY/MM/DD`。
- 提交值仍保持为：`yyyy-MM-dd`（兼容既有表单与接口处理逻辑）。

涉及文件：
- `frontend/components/ui/date-picker.tsx`

---

### 2) 日历月份/年份下拉选择能力增强
- 为 `Calendar` 增加 `captionLayout="dropdown"`。
- 用户可直接通过下拉切换月份与年份，减少逐月翻页操作。

涉及文件：
- `frontend/components/ui/date-picker.tsx`

---

### 3) 组件兼容性修正
- 排查并修复 `PopoverTrigger` 的 `asChild` 使用问题（当前 UI 基础库不支持该属性）。
- 调整后类型检查通过，无新增编译错误。

---

### 4) 表单日期默认值优化
- `productionDate` 默认值改为“今日”。
- `expiryDate` 默认值改为“今日 + 5 年”。

涉及文件：
- `frontend/components/labels/LabelForm.tsx`

---

## 解耦与抽象更新（后置）

### 5) API 路由常量化
- 新增统一路由定义，避免各处硬编码接口字符串。
- 结果：后续接口调整仅需改一处。

涉及文件：
- `frontend/features/labels/api/routes.ts`（新增）
- `frontend/hooks/useLabels.ts`（改造为使用路由常量）
- `frontend/hooks/useLabelHistory.ts`（改造为使用路由常量）

---

### 6) 预览参数构建能力下沉
- 从 UI 组件中抽离 HRI 解析和 preview payload 组装逻辑。
- 统一处理 PI 字段（批号/生产日期/有效期/序列号）与日期归一化。

涉及文件：
- `frontend/features/labels/preview/payload.ts`（新增）

---

### 7) 预览编排副作用下沉（第二批核心）
- 将“SVG 预览请求 + PNG 增强回填 + 错误回退”从弹窗组件移至专用 hook。
- UI 组件只负责状态切换和展示，业务流程交给 orchestrator。

涉及文件：
- `frontend/features/labels/preview/useLabelPreviewOrchestrator.ts`（新增）
- `frontend/components/labels/PreviewDialog.tsx`（重构，移除大段副作用逻辑）

---

### 8) 条码 SVG 渲染工具抽离
- 抽离 `bwip-js` 生成与线性条码 SVG 归一化能力。
- 统一双码模板常量（宽/高），保证可维护性。

涉及文件：
- `frontend/features/labels/preview/barcode-svg.ts`（新增）
- `frontend/components/labels/PreviewTemplateCanvas.tsx`（改为依赖抽象工具）

---

### 9) 导出能力服务化（第三批核心）
- 将 PNG/SVG/PDF 下载逻辑从弹窗组件中彻底抽离为独立服务。
- `PreviewDialog` 仅触发 `exportPreviewNode()`，不再直接依赖导出细节。

涉及文件：
- `frontend/features/labels/preview/export.ts`（新增）
- `frontend/components/labels/PreviewDialog.tsx`（重构下载逻辑）

## 功能与体验层面结果
- 预览组件职责显著收敛：
  - 过去：既负责 UI，又负责数据编排、请求、回退、导出。
  - 现在：主要负责 UI 交互与状态组织。
- 核心能力模块化：
  - 路由、payload、编排、条码 SVG、导出均可独立复用与测试。
- 双码模板 SVG 宽度一致性方案稳定可用（用户已确认“这回对了”）。

## 质量与验证
- 前端 lint 结果：无 error。
- 当前保留 warning（不阻断）：
  - `app/history/page.tsx` 的 hook 依赖提示。
  - 多处 `<img>` 的 next/image 优化建议。

## 兼容性与约定（已延续）
- 日期传输格式：`YY/MM/DD`。
- 条码渲染策略：前端统一渲染（尤其 SVG 路径）。
- 历史记录策略：继续保存 base64 结果。

## 影响范围
- 本次改造聚焦前端，不涉及后端接口协议破坏性变更。
- 主要收益：降低耦合、提高可维护性、为后续“第四批解耦”打基础。

## 下阶段建议（v1.3 方向）
1. 抽离模板 ViewModel（AI 文案/展示字段映射）
2. 继续瘦身页面容器（`app/page.tsx`）
3. 针对 preview/导出模块补充单元测试
4. 逐步处理 lint warnings（`next/image` 与 effect 依赖）
